# Дизайн-специфікація: Сайт адвокатського об'єднання

**Дата:** 2026-05-30
**Тип:** Односторінковий статичний сайт (SPA), мова — українська

## 1. Мета та контекст

Сайт-візитка адвокатського об'єднання (не окремого адвоката) з широкою практикою.
Дві рівноцінні цілі:
1. Залучення клієнтів (зв'язок через прямі контакти).
2. Просування бренду та присутності в соцмережах (Instagram / YouTube / TikTok / Telegram).

## 2. Технічний підхід

- **Чистий статичний сайт**, без збірки та бекенду (Node на машині не встановлено).
- Три файли:
  - `index.html` — семантична розмітка всіх секцій.
  - `styles.css` — стилі, CSS-змінні кольорів/шрифтів, адаптив.
  - `script.js` — інтерактив + **об'єкт-конфіг `SITE_DATA`** зверху файлу з усіма даними (назва, слоган, телефони, email, адреса, посилання на соцмережі, перелік послуг, відгуки, FAQ). Коментарі українською.
- Шрифти Google Fonts: **Playfair Display** (заголовки), **Inter** (текст).
- Відкривається подвійним кліком (`file://`), хоститься на Netlify / GitHub Pages.
- Без зовнішніх залежностей і фреймворків. Іконки соцмереж — вбудований inline SVG.

## 3. Візуальний стиль — «Класична довіра» (Варіант A)

- Фон: темно-синій `#0e1c33`, темніший `#0b1729` для чергування секцій.
- Акцент: золото `#d9b46a`.
- Текст: світлий `#f4f1ea`, приглушений `#c4cdda`.
- Заголовки — Playfair Display; основний текст — Inter.
- Картки: `#13243f` з тонкою рамкою `rgba(255,255,255,.06)`, заокруглення ~12px.
- Кнопки: основна — золота заливка; вторинна — прозора з рамкою.

## 4. Структура сторінки (секції зверху вниз)

1. **Шапка (nav)** — назва об'єднання (логотип-текст) + меню (Про нас, Команда, Послуги, Відгуки, FAQ, Контакти) + кнопка «Консультація». На мобільному — бургер-меню (тогл на JS).
2. **Hero** — назва об'єднання, слоган, підзаголовок, дві кнопки (consultation / послуги), блок статистики (3 показники: роки досвіду, кількість справ, % успіху).
3. **Про нас** — текст про об'єднання + сітка карток «Чому ми» (переваги/цінності).
4. **Команда** — сітка карток адвокатів: фото, ім'я, посада/спеціалізація, коротке біо. Дані з `SITE_DATA.team`, рендеряться JS. Фото — заглушки (монограма/плейсхолдер), власник замінює на справжні.
5. **Послуги / практики** — сітка карток (inline SVG-іконка + назва + опис). Дані з `SITE_DATA.services`, рендеряться JS.
6. **Відгуки клієнтів** — картки-цитати (текст + ім'я). Дані з `SITE_DATA.testimonials`.
7. **FAQ** — акордеон (розгортання питань на JS, одне відкрите за раз). Дані з `SITE_DATA.faq`.
8. **Соцмережі** — великі кнопки-посилання: Instagram, YouTube, TikTok, Telegram (inline SVG-іконки). Посилання з `SITE_DATA.social`.
9. **Контакти** — телефон, email, адреса, графік роботи + кнопки дій (Написати в Telegram / Зателефонувати / Email) + **вбудована Google Maps** (iframe за адресою, без API-ключа).
10. **Футер** — копірайт, дублі ключових посилань і соцмереж.

## 5. Інтерактив (script.js)

- Мобільне бургер-меню (відкрити/закрити, закриття при кліку на пункт).
- Плавний скрол до секцій по якорях.
- Акордеон FAQ.
- **Кнопка «нагору»** — з'являється після прокрутки вниз, плавний скрол до верху.
- Поява секцій при прокрутці (IntersectionObserver, fade/slide-in).
- Рендер карток послуг, відгуків та FAQ з `SITE_DATA`.

## 6. Адаптивність

- Десктоп: багатоколонкові сітки, hero у 2 колонки.
- Планшет: 2 колонки в сітках.
- Мобільний: 1 колонка, бургер-меню, приховані другорядні елементи, читабельні розміри.

## 7. Конфігурація даних (`SITE_DATA`)

Один об'єкт зверху `script.js`, який редагує власник без правок розмітки:
```
SITE_DATA = {
  brand: { name, tagline, subtitle },
  stats: [ {num, label}, ... ],
  about: { text, advantages: [{icon, title, text}] },
  team: [ {name, role, bio, photo}, ... ],
  services: [ {icon, title, text}, ... ],
  testimonials: [ {text, author}, ... ],
  faq: [ {q, a}, ... ],
  social: { instagram, youtube, tiktok, telegram },
  contacts: { phone, email, address, hours, mapEmbedUrl, telegramUrl }
}
```
Усі поля заповнюються прикладами-заглушками; власник замінює на справжні дані.

## 8. Поза межами (YAGNI)

- Блок «Команда» — лише картки (без детальних модальних вікон; можна додати пізніше).
- Немає форми зі сервером — лише прямі контакти.
- Немає живих ембедів соцмереж — лише кнопки-посилання.
- Немає мультимовності, CMS, блогу.
